# 家計簿アプリ
手軽に月の収入と支出を入力、管理することができるアプリです

### 開発の背景
会社の研修として、中途採用の未経験者４人が集まり２週間で家計簿アプリを作成しました。家計簿アプリを作ること、必要最低限の実装機能のみを課題としていただき、技術選定から実装までを自分たちで行いました。

## 使用技術
eclipse: version 4.38.0\
Java: version 21\
Swing: version 1.9.0 \
GSON: version 2.10.1 \
git: 2.53.0.windows.1\
github

## 技術選定理由
今回は全員が未経験ということで、できるだけこれまでに学習していたことが活かせる言語ということでJAVAを選びました。またGUIの部分では、一般的に広く使われている言語であれば、ネット上にも多くの情報があり、詰まった時にも解決策を探しやすいと予測し、フレームワークとしてswingを採用しています。\
データの保存系形式に関しては、今回JSONファイルを使用して保存する、という条件があったので、JacksonかGSONの二つが候補に挙がりましたが、今回のアプリではそこまで複雑な処理をしないアプリだったので、シンプルに使えるGSONを採用しました。

## 工夫した点
### メンバー１【月次サマリ（ホーム画面）担当】
まずは形にするということを念頭に開発に取り組みました。1つのデータを複数人で操作していくにあたって決めておくべきだったことが後手に回り修正が必要になることもありましたが、今回の研修で体感することができたので次の開発につなげていきたいです。また、担当したHOME画面(サマリー表示)では収入と支出を分けて表示させるためにチームで連携し実装しました。拡張機能として、収支を可視化させるため棒グラフの表示にも取り組みました。こちらではプラス収支とマイナス収支をわかりやすくするため、0基準で描画しグラフ全体が見やすく収まるようスケーリング設計をしました。
### メンバー２【入力画面担当】
初めてのチーム開発にほとんど触ったことがない言語でしたので、まずは明確に現状で自分ができることできないことをメンバーに伝えたうえで入力画面を二人で担当させていただきました。
自分の得意な分野であるＵＩの部分を最初に着手してその後ＵＩ部分と動作のコードを繋げていき、少しずつ形にしていくことで理解をしていきました。
二人で担当した画面でしたので、できるだけ読みやすいコードの作成を心掛けるようにしました。

### メンバー３【入力画面担当】
家計簿アプリにおける入力機能の実装を担当しました。
具体的には、日付・金額・カテゴリ・メモを入力し、その内容を JSON形式で保存する処理を作成しました。
工夫した点として、以下の点を意識しました。
• 支出と収入を明確に区別できるように設計
支出の場合は金額をマイナスの値として保存することで、後続の集計処理や表示処理がしやすくなるようにしました。
• 入力値のバリデーションを実装
金額入力では、0〜9の数字以外が入力されないようチェック処理を追加し、想定外の値が保存されるのを防ぎました。
チーム開発を意識した可読性の確保
　案件が決まっており抜けてしまうので、残りの工程をお願いするので、変数名や処理の流れが他のメンバーにも分かりやすくなるよう意識し、後から見ても理解しやすいコードを書くことを心がけました。

### メンバー４【一覧画面担当】
まず初めてのチーム開発ということで、どのように役割分担をしていくのかが課題でした。最初はコミュニケーションの量を増やしチームメンバへの理解や言語への理解を深め、後半はそれぞれの得意な範囲も理解できてきたので、毎日のミーティングで自分の担当タスクを伝え合い、間に合ってない部分や、抜けがある部分はフォローしながら実装できたことは、今後の業務でチーム開発する際にも活かせそうだと感じました。
実装部分に関しては、一覧取得を主に担当しましたが、コードを書く際に可読性を意識し、変数名などもほかの人が見たときに分かりやすくなるよう心掛けました。
## 課題＆今後の取り組み

GitHubの使用方法を明瞭にすればもう少し円滑にチーム開発を行えたかなと思うので、小規模の開発を多数行って慣れていきたいと考えています。また、コーディングの内容について伝えたり、教えたりする際にうまく説明できないこともあったのでインプットとアウトプットを繰り返し行い、チーム開発における円滑なコミュニケーション力もつける必要があると感じました。


### メンバー２【入力画面担当】
今回は「アプリが動く」ということを目標に実装を進めました。
Javaの知識不足などでエラーが複数起こり「アプリが動かない」という状況が頻発したので次チーム開発をするときまでには、
Javaの基礎知識をしっかりと定着させたうえで、コードの作成を行えるようになりたいと思います。

### メンバー３【入力画面担当】
入力チェックの拡張余地
数値チェックは行いましたが、日付の範囲チェックや未入力時のエラー出力、入力エラーなどユーザー視点を意識することが大切だと感じました。

### メンバー４【一覧画面担当】
今回は「まず、動く」という点を目標に実装を進めたので、オブジェクト指向や、メソッドの切り出し、クラスごとに分けるといったことは効率的にできませんでした。
また一番初めにデータ保持の型をしっかりと決めずに始めてしまったので、実装中に不具合が頻発し、コード修正を余儀なくされたので、次チーム開発をするときはまずデータの保持の仕方から話し合っていきたいと思います。
